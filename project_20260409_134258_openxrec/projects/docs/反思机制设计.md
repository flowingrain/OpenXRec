# 反思机制（Reflection Mechanism）

## 设计理念

反思机制为推荐系统引入"元认知"能力，使系统在决策后能够自我评估、反事实推理和生成改进建议。

```
决策层 → 执行层 → 反思层 → 优化层
   ↓        ↓        ↓        ↓
 意图判断  推荐生成  质量评估  参数调整
          执行变更  风险评估  回滚决策
          训练模型  效果评估  采纳决策
```

## 核心价值

| 价值 | 说明 |
|------|------|
| **自我评估** | 对决策质量进行量化评估，识别优点和不足 |
| **反事实推理** | 思考"如果选择不同会怎样"，识别潜在更好方案 |
| **改进建议** | 基于分析生成可执行的改进建议 |
| **可解释性增强** | 提供推理链和关键因素解释 |

## 反思结果结构

```typescript
interface ReflectionResult {
  // 自我评估
  selfEvaluation: {
    score: number;               // 自评分数 0-1
    strengths: string[];         // 优点
    weaknesses: string[];        // 不足
  };
  
  // 反事实推理
  counterfactualAnalysis: {
    alternativeDecisions: Array<{
      decision: string;          // 替代决策
      expectedOutcome: string;   // 预期结果
      probability: number;       // 可能性
    }>;
    bestAlternative?: string;    // 最佳替代方案
    reasoning: string;           // 推理过程
  };
  
  // 改进建议
  improvements: Array<{
    area: string;                // 改进领域
    suggestion: string;          // 具体建议
    priority: 'high' | 'medium' | 'low';
    implementable: boolean;      // 是否可实施
  }>;
  
  // 可解释性
  explainability: {
    reasoningChain: string[];    // 推理链
    keyFactors: string[];        // 关键因素
    confidenceExplanation: string;
  };
}
```

## 使用场景

### 1. 意图分析反思

```typescript
const reflectionService = getReflectionService();

// 意图分析后反思
const reflection = await reflectionService.reflectOnIntentAnalysis({
  query: '2024年最新的AI发展趋势',
  scenario: 'tech_trend',
  predictedNeedsSearch: true,
  predictedConfidence: 0.85,
  reasoning: '查询包含时间词和"最新"关键词',
  matchedKeywords: ['2024', '最新'],
  source: 'llm',
});

// 根据反思调整决策
if (reflection.selfEvaluation.score < 0.6) {
  // 降低置信度或触发额外验证
  confidence *= 0.7;
}
```

### 2. 规则更新反思

```typescript
// 规则更新前反思
const result = await reflectionService.reflectOnRuleUpdate({
  updateType: 'keyword_add',
  beforeState: { keyword: null, category: null },
  afterState: { keyword: '如何', category: 'need_search' },
  triggerReason: '新发现的高频关键词',
  sampleEvidence: [
    { query: '如何学习Python', predicted: true, actual: true },
    { query: '如何注册账号', predicted: true, actual: true },
  ],
});

// 根据反思决定是否执行
if (!result.shouldProceed) {
  console.log('反思分析不建议执行此更新:', 
    result.selfEvaluation.weaknesses);
  return;
}
```

### 3. PPO训练反思

```typescript
// PPO训练后反思
const result = await reflectionService.reflectOnPPOTraining({
  epochsTrained: 10,
  samplesUsed: 256,
  policyLoss: 0.15,
  valueLoss: 0.08,
  avgReward: 2.3,
  improvement: 0.12,
  strategyWeightsBefore: { content_based: 0.3, collaborative: 0.3 },
  strategyWeightsAfter: { content_based: 0.35, collaborative: 0.25 },
});

// 根据反思决定是否采纳新策略
if (!result.isImproving) {
  console.log('检测到潜在问题，回滚策略');
  await rollbackPolicy();
}
```

## API接口

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/v1/reflection` | GET | 获取反思历史和报告 |
| `/api/v1/reflection` | POST | 触发反思分析 |

### GET示例

```bash
# 获取反思历史
curl http://localhost:5000/api/v1/reflection?action=history&limit=20

# 获取反思报告
curl http://localhost:5000/api/v1/reflection?action=report

# 批量反思分析
curl http://localhost:5000/api/v1/reflection?action=batch
```

### POST示例

```bash
# 触发意图分析反思
curl -X POST http://localhost:5000/api/v1/reflection \
  -H 'Content-Type: application/json' \
  -d '{
    "type": "intent",
    "input": {
      "query": "2024年最新的AI发展趋势",
      "scenario": "tech_trend",
      "predictedNeedsSearch": true,
      "predictedConfidence": 0.85,
      "reasoning": "查询包含时间词",
      "matchedKeywords": ["2024", "最新"],
      "source": "llm"
    }
  }'
```

## 批量反思分析

定期执行批量反思以发现系统性问题：

```typescript
const batchResult = await reflectionService.performBatchReflection({
  recentFeedbackCount: 100,
});

console.log('平均自评分数:', batchResult.summary.avgSelfScore);

```

## 与现有系统的集成

### 与自适应优化集成

```
反馈收集 → 规则更新请求 → 反思评估 → 执行/拒绝
                              ↓
                        记录反思结果
                              ↓
                        用于后续优化
```

### 与PPO优化集成

```
PPO训练 → 策略变更 → 反思评估 → 采纳/回滚
                            ↓
                      记录反思结果
                            ↓
                      调整训练参数
```

### 与持久化系统集成

反思结果自动持久化到 Supabase：

```sql
-- 反思记录表
CREATE TABLE reflection_records (
  id SERIAL PRIMARY KEY,
  trigger VARCHAR(50),
  self_score FLOAT,
  strengths TEXT[],
  weaknesses TEXT[],
  improvements JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 最佳实践

1. **关键决策必反思**：意图分析、规则更新、策略变更等关键决策后都应进行反思
2. **低置信度强制反思**：当决策置信度低于阈值时，强制触发反思
3. **批量分析定期执行**：每小时或每天执行批量反思分析
4. **反思结果可视化**：在仪表盘中展示反思分数趋势
5. **反思驱动优化**：将反思建议作为系统优化的输入
