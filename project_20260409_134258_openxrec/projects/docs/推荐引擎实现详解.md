### 2.2 推荐流程（8步）

```typescript
async recommend(request: RecommendationRequest): Promise<RecommendationResult[]> {
  // Step 1: 获取用户画像
  const userProfile = await this.getUserProfile(userId);
  
  // Step 2: 检查缓存
  if (this.config.enableCache) {
    const cached = this.cache.get(cacheKey);
    if (cached && !expired) return cached.results;
  }
  
  // Step 3: 确定推荐策略
  const effectiveStrategy = strategy || this.selectStrategy(userProfile, scenario, context);
  
  // Step 4: 获取候选物品
  let candidates = await this.getCandidates(userProfile, scenario, filters, context);
  
  // Step 5: 根据策略进行评分
  let scoredItems = await this.scoreItems(candidates, userProfile, effectiveStrategy);
  
  // Step 6: 过滤低分项
  scoredItems = scoredItems.filter(s => s.score >= minScoreThreshold);
  
  // Step 7: 排序和多样性优化
  const rankedItems = await this.rankingWithOptimization(scoredItems, userProfile, options);
  
  // Step 8: 生成解释
  const results = [];
  for (const { item, score, details } of rankedItems) {
    const explanations = await this.generateExplanations(item, userProfile, score, details);
    results.push({ item, score, rank, explanations, confidence, strategy, agents });
  }
  
  return results;
}
```

## 三、6种推荐策略详解

### 3.1 基于内容的推荐 (Content-based)

**原理**: 推荐与用户历史喜好相似的物品

**评分逻辑**:
```typescript
private async contentBasedScoring(candidates, userProfile) {
  for (const item of candidates) {
    let score = 0;
    
    // 1. 兴趣匹配度 (30%)
    for (const interest of userProfile.interests) {
      if (item.description?.includes(interest) || item.title?.includes(interest)) {
        score += 0.3;
      }
    }
    
    // 2. 属性相似度 (20%)
    for (const [key, value] of Object.entries(item.attributes)) {
      if (userProfile.preferences[key] === value) {
        score += 0.2;
      }
    }
    
    // 3. 向量相似度 (50%)
    if (userProfile.embeddings && item.embeddings) {
      const similarity = cosineSimilarity(userProfile.embeddings, item.embeddings);
      score += similarity * 0.5;
    }
  }
}
```

**优点**: 
- 无需其他用户数据
- 可解释性强
- 解决冷启动问题

**缺点**: 
- 只能推荐相似物品
- 缺乏惊喜性

---

### 3.2 协同过滤推荐 (Collaborative Filtering)

**原理**: 基于相似用户的行为进行推荐

**评分逻辑**:
```typescript
private async collaborativeScoring(candidates, userProfile) {
  // 1. 找到相似用户
  const similarUsers = await this.findSimilarUsers(userProfile, 10);
  
  // 2. 聚合相似用户对物品的评分
  for (const item of candidates) {
    let score = 0;
    let voteCount = 0;
    
    for (const similarUser of similarUsers) {
      const behavior = similarUser.behaviorHistory.find(b => b.itemId === item.id);
      if (behavior && (behavior.action === 'like' || behavior.action === 'purchase')) {
        score += behavior.rating ? behavior.rating / 5 : 1;
        voteCount++;
      }
    }
    
    if (voteCount > 0) {
      score = score / voteCount;
    }
  }
}
```

**优点**:
- 能发现用户未接触过的物品
- 推荐质量高

**缺点**:
- 冷启动问题
- 数据稀疏性
- 隐私问题

---

### 3.3 混合推荐 (Hybrid)

**原理**: 结合多种策略，取长补短

**评分逻辑**:
```typescript
private async hybridScoring(candidates, userProfile) {
  // 并行计算多个策略的分数
  const [contentScores, collaborativeScores, knowledgeScores] = await Promise.all([
    this.contentBasedScoring(candidates, userProfile),
    this.collaborativeScoring(candidates, userProfile),
    this.knowledgeBasedScoring(candidates, userProfile)
  ]);
  
  // 加权组合
  const combinedScore =
    content.score * 0.3 +          // 基于内容 30%
    collaborative.score * 0.3 +   // 协同过滤 30%
    knowledge.score * 0.2 +       // 知识图谱 20%
    agent.score * 0.1 +           // 智能体 10%
    causal.score * 0.1;           // 因果推断 10%
}
```

**优点**:
- 综合多种策略优势
- 推荐质量最高

### 3.4 知识图谱驱动推荐 (Knowledge-based)

**原理**: 利用知识图谱推理隐式关联

**评分逻辑**:
```typescript
private async knowledgeBasedScoring(candidates, userProfile) {
  for (const item of candidates) {
    let score = 0;
    
    // 1. 检查物品属性与知识图谱节点的关联
    for (const node of userProfile.knowledgeGraph.nodes) {
      if (node.type === 'interest' && item.attributes.some(attr => 
          String(attr).includes(node.label))) {
        score += node.weight * 0.3;
      }
    }
    
    // 2. 检查知识图谱中的关系
    for (const edge of userProfile.knowledgeGraph.edges) {
      if (item.attributes[edge.relation] === edge.target) {
        score += edge.weight * 0.2;
      }
    }
  }
}
```

**优点**:
- 可解释性强
- 能发现隐式关联
- 跨领域推荐

### 3.5 智能体驱动推荐 (Agent-based) ⭐

**原理**: 使用多智能体系统进行推理和评分

**智能体架构**:
```
感知层 (Perception Layer)
├── 用户画像智能体 (user_profiler)
└── 物品画像智能体 (item_profiler)

认知层 (Cognition Layer)
├── 特征提取智能体 (feature_extractor)
├── 相似度计算智能体 (similarity_calculator)
├── 因果推理智能体 (causal_reasoner)
└── 知识图谱推理智能体 (kg_reasoner)

决策层 (Decision Layer)
├── 排序智能体 (ranking_agent)
├── 解释生成智能体 (explanation_generator)
├── 多样性优化智能体 (diversity_optimizer)
└── 新颖性检测智能体 (novelty_detector)
```

**评分逻辑**:
```typescript
private async agentBasedScoring(candidates, userProfile, context) {
  for (const item of candidates) {
    // 调用多智能体系统
    const agentScores = await this.callAgentScoring(item, userProfile, context);
    
    // 聚合多个智能体的评分
    const score = agentScores.reduce((sum, s) => sum + s.score, 0) / agentScores.length;
    
    return {
      score,
      details: {
        factors: agentScores.map(s => ({
          name: `agent_${s.agentName}`,
          value: s.reasoning,      // 智能体的推理过程
          importance: s.score
        })),
        agents: agentScores.map(s => s.agentName)
      }
    };
  }
}
```

**优点**:
- 推理过程透明
- 可解释性最强
- 灵活性高
- 能处理复杂场景

### 3.6 因果推断驱动推荐 (Causal-based) ⭐

**原理**: 分析推荐结果的因果影响，确保推荐的有效性

**评分逻辑**:
```typescript
private async causalBasedScoring(candidates, userProfile, context) {
  for (const item of candidates) {
    // 调用因果推断引擎
    const causalScore = await this.callCausalInference(item, userProfile, context);
    
    return {
      score: causalScore.score,
      details: {
        factors: [{
          name: 'causal_impact',
          value: causalScore.causalChain,  // 因果链: 兴趣匹配 → 行为倾向 → 推荐结果
          importance: causalScore.score
        }],
        agents: ['causal_analyst']
      }
    };
  }
}
```

**优点**:
- 推荐更可靠
- 避免虚假相关
- 可预测影响

```typescript
private async generateExplanations(item, userProfile, score, details, strategy) {
  const explanations = [];
  
  // 基于特征相似性的解释
  if (details.factors) {
    const reason = `基于你的兴趣和偏好：${details.factors
      .slice(0, 3)
      .map(f => f.value)
      .join('、')}`;
    explanations.push({
      type: 'feature_similarity',
      reason,
      factors: details.factors.slice(0, 5),
      weight: 0.4
    });

  // 使用 LLM 生成更自然的解释
```

```typescript

export async function userProfilerNode(state, llmClient) {
  // LLM 是智能体的核心决策引擎

  const response = await llmClient.chat([

  ]);
  
  // 解析 LLM 返回的结果
```

```typescript
async function recommendWithAgents(request) {
  const stream = new ReadableStream({
    async start(controller) {
      const llmClient = new LLMClient();
      const initialState = { userId, items, context, options };
      
      // 流式执行智能体工作流
      for await (const event of recommendationGraph.stream(initialState, config)) {
        // 实时推送每个智能体的执行进度
        const update = {
          type: 'progress',
          node: Object.keys(event)[0],  // 当前执行的智能体
          timestamp: Date.now()
        };
        controller.enqueue(encoder.encode(`data: ${JSON.stringify(update)}\n\n`));
      }
      
      // 获取最终结果
      const finalState = await recommendationGraph.invoke(initialState, config);
      
      // 推送最终结果
      controller.enqueue(encoder.encode(`data: ${JSON.stringify({
        type: 'complete',
        results: finalState.results
      })}\n\n`));
    }
  });
  
  return new Response(stream, {
    headers: { 'Content-Type': 'text/event-stream' }
  });
}
```

1. **感知层**: 用户画像、物品画像
