# 用户系统与可解释推荐系统 - 实现文档

## 概述

本次实现为可解释推荐系统添加了完整的用户系统，包括用户注册、登录、交互历史记录和用户画像生成功能。

## 已完成的功能

### 1. 数据库设计

#### 新增表结构

1. **users** - 用户表
   - 用户基本信息（邮箱、用户名、密码）
   - 用户状态（是否活跃、邮箱验证）
   - 角色管理（用户、管理员）

2. **user_sessions** - 用户会话表
   - 会话令牌管理
   - 会话过期时间
   - IP 地址和用户代理记录

3. **user_interactions** - 交互历史表
   - 13种交互类型（view、click、like、dislike等）
   - 5种内容类型（推荐、知识、实体、关系、案例）
   - 详细的上下文和元数据记录

4. **user_profiles** - 用户画像表
   - 兴趣偏好分析
   - 行为模式识别
   - 推荐偏好权重
   - 交互统计数据

5. **recommendation_history** - 推荐历史表
   - 推荐记录
   - 算法和参数
   - 用户反馈和点击率

6. **user_feedback** - 用户反馈表
   - 6种反馈类型
   - 评分和评论
   - 情感分析

### 2. API 接口

#### 用户认证相关

- `POST /api/auth/register` - 用户注册
- `POST /api/auth/login` - 用户登录
- `GET /api/auth/me` - 获取当前用户信息
- `DELETE /api/auth/me` - 用户登出

#### 交互历史相关

- `GET /api/interactions` - 获取用户交互历史
- `POST /api/interactions` - 记录用户交互
- `GET /api/interactions/stats` - 获取用户统计

#### 用户画像相关

- `GET /api/profile` - 获取用户画像
- `POST /api/profile` - 生成/更新用户画像
- `PATCH /api/profile` - 更新用户画像权重

### 3. 前端组件

1. **AuthContext** - 认证上下文
   - 提供全局认证状态管理
   - 自动处理令牌存储和刷新

2. **AuthPage** - 登录/注册页面
   - 表单验证
   - 错误处理
   - 加载状态

3. **ProfilePage** - 用户画像页面
   - 画像数据展示
   - 兴趣偏好可视化
   - 推荐权重调整

### 4. 服务层

1. **UserService** - 用户服务
   - 用户注册/登录
   - 密码哈希和验证
   - JWT 令牌生成和验证

2. **InteractionService** - 交互服务
   - 记录用户交互
   - 查询交互历史
   - 用户统计分析

3. **UserProfileService** - 用户画像服务
   - 画像生成和分析
   - 行为模式识别
   - 偏好权重计算

```typescript
  return (
    <html lang="zh-CN">
      <body>
        <AuthProvider>
          {children}
        </AuthProvider>
      </body>
    </html>
  );

```

#### 使用认证功能

```typescript
import { useAuth } from '@/contexts/AuthContext';

function MyComponent() {
  const { user, login, logout } = useAuth();

  if (!user) {
    return <button onClick={() => login({...)}>登录</button>;
  }

  return (
    <div>
      <p>欢迎，{user.username}</p>
      <button onClick={logout}>登出</button>
    </div>
  );
}
```

```typescript
// 获取用户画像
async function getUserProfile() {
  const token = localStorage.getItem('token');
  const response = await fetch('/api/profile', {
    headers: {
      'Authorization': `Bearer ${token}`,
    },
  });

  const result = await response.json();
  if (result.success) {
    console.log('用户画像:', result.data);
  }

  const token = localStorage.getItem('token');
  const response = await fetch('/api/profile', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
    },
  });

  if (result.success) {
    console.log('新画像已生成:', result.data);
  }

```

## 用户画像生成逻辑

用户画像系统会自动分析用户的交互历史，提取以下特征：

### 1. 兴趣偏好

- 基于交互内容的分类和实体
- 计算各类别的交互频率
- 识别用户偏好的主题

### 2. 行为模式

- **查看点击比**：衡量用户从查看到点击的转化率
- **点赞差评比**：衡量用户对内容的喜好程度
- **平均会话时长**：衡量用户的使用粘性
- **活跃时段**：识别用户最活跃的时间段

### 3. 推荐权重

根据用户行为自动调整三个权重：

- **多样性权重**：控制推荐结果的多样性
- **新颖性权重**：控制推荐新内容的倾向
- **相关性权重**：控制推荐内容与历史的相关性

### 4. 时间模式

分析用户在一天中不同时间段的活跃情况：
- 早晨（5-12点）
- 下午（12-18点）
- 傍晚（18-22点）
- 深夜（22-5点）

## 安全特性

1. **密码哈希**：使用 SHA-256 哈希存储密码
2. **JWT 令牌**：使用 JWT 进行身份验证
3. **会话管理**：支持会话过期和清理
4. **行级安全**：使用 RLS 策略保护数据
5. **令牌验证**：每个 API 请求都验证令牌有效性

## 后续集成建议

### 1. 与推荐系统集成

在推荐算法中使用用户画像：

```typescript
// 获取用户画像
const profile = await UserProfileService.getUserProfile(userId);

// 使用画像权重调整推荐
const recommendations = await recommendationEngine.generate({
  userId,
  preferences: {
    diversity: profile.diversity_weight,
    novelty: profile.novelty_weight,
    relevance: profile.relevance_weight
  },
  interests: profile.interests
});
```

### 2. 自动记录交互

在推荐结果页面自动记录交互：

```typescript
// 页面加载时记录查看
useEffect(() => {
  recordView(recommendationId);
}, [recommendationId]);

// 点击时记录
const handleClick = () => {
  recordClick(recommendationId);
  // 处理点击逻辑
};
```

### 3. 定期更新画像

设置定时任务或在关键操作后更新用户画像：

```typescript
// 每天自动更新画像
setInterval(async () => {
  if (user) {
    await generateProfile();
  }
}, 24 * 60 * 60 * 1000);

// 用户完成特定操作后更新
const onUserCompleteAction = async () => {
  // ... 执行操作
  await generateProfile();
};
```
