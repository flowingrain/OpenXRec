# 架构调整说明：目标管理的职责拆分

## 背景问题

原架构中将"目标管理"放在**感知体层**存在概念矛盾：

| 感知体层职责 | 目标管理职责 |
|-------------|-------------|
| 信息采集（从外部获取数据） | 任务理解（决定做什么） |
| 环境监控（检测变化） | 目标分解（规划路径） |
| 数据预处理 | 工作流规划 |
| **自底向上** | **自顶向下** |

## 解决方案：职责拆分

将原"目标管理"拆分为两个智能体，分别放在不同层级：

### 1. 意图解析器 (intent_parser) - 调度层（独立协调器）

**职责**：
- 快速理解用户意图
- 确定基本信息需求
- 判断任务复杂度
- 决定初步执行策略
- 智能体调度与资源分配

**特点**：协调器角色，独立于智能体

### 2. 战略规划器 (strategy_planner) - 决策体层

**职责**：
- 基于认知结果进行深度分析规划
- 目标分解与任务细化
- 分析方向调整
- 策略优化建议

**特点**：基于认知结果、深度规划

## 调度流程说明

### 调整前

```
goal_manager(感知体) → 决定复杂度和跳过策略 → 其他智能体执行
```

**问题**：规划在感知之前，概念矛盾

### 调整后

```
intent_parser(感知体) → 快速意图理解 → 初步复杂度判断
        ↓
其他感知体 → 信息采集
        ↓
认知体 → 分析推理
        ↓
strategy_planner(决策体) → 基于认知结果深度规划 → 调整后续分析
        ↓
其他决策体 → 场景推演、敏感性分析
```

**优势**：
1. 意图解析在感知阶段，快速响应
2. 深度规划在有认知结果后，更加精准
3. 职责清晰，符合认知科学原理
