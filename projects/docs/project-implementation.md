# 上市公司风险分析智能体系统 - 项目方案文档

> 基于"感知-认知-决策-行动-进化"五体智能体架构，重点攻关**因果推断**与**事件图谱**两大核心技术，通过"**数字人格 + 因果推理**"双引擎模式，实现风险的可解释分析与可推演决策，构建具备自进化能力的经济风险智能体系统。

---

## 核心定位

**闭环能力（目标链路）**：
```
多源数据 → 事件抽取 → 因果推理 → 风险推演 → 决策输出 → 持续进化
    ↓          ↓          ↓          ↓          ↓          ↓
  感知体     认知体     决策体     行动体     进化体
```

**核心技术路径**：
```
数字人格（表达层）+ 因果推断（推理层）+ 多智能体协同（执行层）
```

**工程栈**：
- 演示验证：Coze 平台（Next.js 16 / React 19、LangGraph JS、图谱可视化）
- 后续主实现：迁移至 Python 本地开发环境，因果/图谱/推演能力服务化

---

## 一、应用背景与案例

### 1.1 业务背景：风险难"早现"、难"解释"、难"联动"

在授信、投资与监管合规等场景中，上市公司风险常见痛点：

| 痛点 | 具体表现 | 业务影响 |
|------|---------|---------|
| **风险难早现** | 风险信号分散在公告、财报、新闻、舆情、交易等多源渠道 | 错过最佳干预时机 |
| **识别不稳定** | 文本噪声与口径差异导致事件抽取不准确 | 误报/漏报频发 |
| **因果难解释** | 单点事件很难说明"为什么发生、如何传导、何时触发" | 决策缺乏依据 |
| **联动缺失** | 预警到研判报告、监控与处置策略之间缺少统一推理链 | 执行效率低 |

### 1.2 案例定义：端到端"上市风险监测"演示链路

```
多源数据接入 → 风险事件抽取（结构化） → 因果推断与传导链重建 
    → 情景推演与预警分级 → 生成研判报告与策略建议 → 沉淀案例并持续进化
```

**核心价值**：
- 因果推断：把"相关信息"转为"可解释的风险传导机制"
- 预警输出：把推理结果转化为业务可执行的监控项与处置优先级

### 1.3 典型输入与输出

#### 输入

| 输入项 | 说明 | 示例 |
|--------|------|------|
| 企业标识 | 企业名称/代码 | "某科技公司" / "000001.SZ" |
| 风险维度 | 目标分析维度 | 财务/治理/合规/市场/ESG |
| 时间窗口 | 分析时间范围 | 近30天/近一季度 |
| 业务约束 | 特定约束条件 | 关注质押风险/监管合规 |

#### 输出

| 输出项 | 说明 | 业务价值 |
|--------|------|---------|
| 动态事件图谱 | 实体-事件-因果关系可视化 | 全局态势感知 |
| 因果风险传导链 | 从事件到风险维度的因果路径 | 可解释推理 |
| 风险预警分级 | Low/Medium/High/Critical | 优先级排序 |
| 研判报告 | 可审计的分析报告 | 合规存档 |
| 策略建议 | 监控项、推荐动作、触发条件 | 可执行指令 |

## 二、业务判断能力详解

### 2.1 输出特征

本系统的"业务判断"不止给结论，而是给出业务可直接落地的决策要素：

| 能力维度 | 说明 | 示例输出 |
|---------|------|---------|
| **风险链路解释** | 从"已观测事件"沿因果边解释到"风险维度" | `事件A → 因果机制 → 风险维度B` |
| **触发条件与时点** | 给出触发阈值/时间窗口，变成可监控规则 | 未来X天内满足条件C |
| **What-if干预预测** | 基于SCM + 干预分析预测策略效果 | 若执行策略S，风险High→Medium |
| **推荐动作** | 映射为授信/监控/处置等可执行动作 | 按优先级排序的动作清单 |
| **置信度与校验** | 输出证据覆盖度与置信度，抑制"幻觉" | 证据来源清单、置信度评分 |

### 2.3 数字人格差异化输出

数字人格通过"约束偏好"改变输出的排序与侧重点，而非仅改变措辞：

| 人格 | 主要约束/偏好 | 输出侧重 | 典型动作组合 |
|------|--------------|---------|-------------|
| **CEO** | 合规前提 + 稳健增长 | 稳增长与兜底动作优先 | 信息披露优化 + 风险兜底方案 |
| **控股股东** | 控制权稳定 + 融资可得性 | 质押/融资/替代路径触发优先 | 补充质押 + 融资替代方案 |
| **机构投资者** | 风险预算 + 止损节奏 | 阈值、回撤、减仓执行条件优先 | 止损线设置 + 减仓节奏规划 |
| **监管机构** | 合规性 + 市场稳定 | 违规风险与投资者保护优先 | 问询函关注 + 处罚风险评估 |
| **审计师** | 审计风险 + 职业声誉 | 审计意见影响与披露风险 | 审计调整建议 + 披露完善 |
| **债权银行** | 贷款安全 + 不良率控制 | 抵押物价值与还款能力 | 授信额度调整 + 增信要求 |

### 2.4 业务价值验收口径

| 价值点 | 验收方式 | 指标示例 |
|--------|---------|---------|
| **减少人工解释成本** | 因果链 + 触发条件替代纯文本研判 | 分析时间缩短 40% |
| **提升一致性** | 标准化输出模板 | 不同分析师结论一致性 > 85% |
| **降低误报风险** | 证据覆盖度/置信度约束 | 误报率下降 30% |
| **提升可审计性** | 证据来源清单 + 可复算依据 | 审计通过率 100% |

## 三、研究内容与关键问题梳理

### 3.1 总体研究目标

构建面向"领域风险分析"的多智能体系统（以上市公司为案例），实现从多源数据到持续进化的闭环。

### 3.2 五体智能体：研究内容、关键问题与实现对照

#### （1）感知体（数据 → 事件）

**研究内容**：

| 方向 | 内容 |
|------|------|
| 多源数据接入 | 监管披露、公告、财报、新闻舆情、交易/运营数据 |
| 事件抽取 | 时间/主体/行为/对象/影响（五元组结构化） |
| 噪声过滤 | 数据清洗、信源可信度评估 |

**关键问题**：

| 多源冲突（真伪/偏见）处理 | ⭐⭐⭐ | 简单去重 | 可信度加权融合，信源评级 |
| 实时性与成本平衡 | ⭐⭐ | 基础实现 | 增量更新 + 缓存策略 |

### 4.3 多智能体协同机制

**研究侧**：任务分解与编排、多模型一致性、异步协同

**落地侧**：

| 复杂度 | 执行策略 | 跳过节点 | 最大迭代 |
|--------|---------|---------|---------|
| simple | 感知→行动 | sensitivity, validator, arbitrator | 1 |
| moderate | 感知→认知→决策→行动 | arbitrator | 2 |
| complex | 完整五层 | 无 | 3 |

**质量保障流程**：

```
分析师输出 → 验证者检查 → 通过？
                │
                ↓ 否
           仲裁者介入
                │
                ↓
         返回修正（最多3次）

质量标准：
├── 完整性：维度覆盖 ≥ 90%
├── 准确性：因果置信度 ≥ 70%
├── 一致性：逻辑无矛盾
└── 可执行性：建议可落地
```

**后续规划**：业务异步总线、多模型投票策略

### 4.6 领域适配（上市公司风险分析）

**六大风险维度**：

| 维度 | 核心指标 | 风险传导路径 |
|------|---------|-------------|
| 财务风险 | Altman Z-Score、资产负债率、流动比率 | 高负债→流动性危机→债务违约 |
| 经营风险 | 毛利率、ROA/ROE、客户集中度 | 毛利率下滑→盈利恶化→估值下调 |
| 治理风险 | 股权质押比例、独董占比、关联交易 | 高质押→股价下跌→控制权危机 |
| 市场风险 | Beta系数、股价波动率、融券余额 | 波动加剧→质押预警→平仓压力 |
| 合规风险 | 监管处罚、诉讼案件、信披评级 | 处罚→声誉受损→融资受限 |
| ESG风险 | ESG评级、碳排放强度、董事会多样性 | ESG下调→机构减持→估值压制 |

**预置风险传导模式**：12+典型传导链，支持动态发现
