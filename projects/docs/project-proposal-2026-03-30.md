# 上市公司风险分析智能体 — 项目方案文档

**文档日期**：2026年3月30日

---

## 核心定位

本项目以「上市公司风险分析」为试点，围绕「感知—认知—决策—行动—进化」五体智能体架构，重点攻关**因果推断**与**事件图谱**两大核心技术，通过「**数字人格 + 因果推理**」双引擎，实现风险的可解释分析与可推演决策，构建具备自进化能力的经济风险智能体系统。

### 闭环能力（目标链路）

```
多源数据 → 事件抽取 → 因果推理 → 风险推演 → 决策输出 → 持续进化
```

### 核心技术路径

```
数字人格（表达层）+ 因果推断（推理层）+ 多智能体协同（执行层）
```

### 当前工程栈

本阶段演示验证基于 Coze 平台完成（原型采用 Next.js 16 / React 19、LangGraph JS、图谱可视化与相关服务）；后续主实现将迁移到 Python 本地开发环境，并将因果/图谱/推演能力服务化，接口对应 `/api/causal/discover|infer|policy` 等。

## 一、应用背景与案例（上市风险监测）

### 1.1 业务背景：风险难"早现"、难"解释"、难"联动"

在授信、投资与监管合规等场景中，上市公司风险常见痛点包括：

| 痛点 | 描述 | 业务影响 |
|------|------|----------|
| **风险难早现** | 风险信号分散在公告、财报、新闻、舆情与交易等多源渠道 | 信息孤岛导致预警滞后 |
| **识别不稳定** | 文本噪声与口径差异导致"识别不稳定" | 误报/漏报率高 |
| **因果难解释** | 风险演化具备因果链条，单点事件很难说明"为什么发生、如何传导、何时触发" | 决策缺乏依据 |
| **联动困难** | 从预警到研判报告、再到监控与处置策略之间缺少统一、闭环的推理决策链 | 业务流程断裂 |

### 1.2 案例定义：端到端"上市风险监测"演示链路

本汇报以"上市风险监测"为案例载体，将链路明确为：

```
多源数据接入 → 风险事件抽取（结构化） → 因果推断与传导链重建 
→ 情景推演与预警分级 → 生成研判报告与策略建议 → 沉淀案例并持续进化
```

其中：
- **因果推断**用于把"相关信息"转为"可解释的风险传导机制"
- **预警输出**用于把推理结果转化为业务可执行的监控项与处置优先级

### 1.3 典型输入与输出

#### 输入
- 企业名称/代码或关键词查询
- 目标风险维度（财务/治理/合规/市场/ESG 等）
- 时间窗口与业务约束

#### 输出
- 动态事件图谱
- 因果风险传导链
- 风险预警分级（可映射 `Low/Medium/High/Critical`）
- 可审计的研判报告与策略建议（监控项、推荐动作与触发条件）

## 二、业务判断能力详解

### 2.1 输出特征

本系统的"业务判断"不止给结论，而是给出业务可直接落地的决策要素：

| 输出要素 | 说明 | 业务价值 |
|----------|------|----------|
| **风险链路解释** | 从"已观测事件"沿因果边解释到"风险维度"，并标注关键证据节点 | 便于合规审计 |
| **触发条件与时点** | 给出触发阈值/时间窗口，把预警变成可监控规则 | 自动化监控 |
| **What-if 干预预测** | 基于 SCM + 干预分析回答"若采取某策略，风险等级预计如何变化、变化幅度与不确定性如何" | 策略模拟 |
| **推荐动作（可执行）** | 把推演结果映射为"授信收紧/追加监控/处置优先级/信息披露关注点"等，并按优先级排序 | 直接执行 |
| **置信度与校验** | 输出证据覆盖度与置信度，抑制"幻觉式推演"，并保留可复算依据 | 可审计性 |

### 2.2 输出模板（业务口播示例）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【风险研判报告】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

风险链路：事件A -> 因果机制 -> 风险维度B

预警分级：High

关键触发条件：未来 X 天内满足 条件C

What-if 结论：
若执行 策略S，风险预计 High -> Medium
（附不确定性区间）

推荐动作（按重要性排序）：
1. 追加监控指标：XXX
2. 授信收紧：XXX
3. 处置优先级：XXX

证据与审计：
- 事件图谱节点清单
- 证据来源清单
- 置信度评分

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 2.3 数字人格差异化输出

数字人格通过"约束偏好"改变输出的排序与侧重点，而不是只改变措辞。

| 人格 | 主要约束/偏好 | 输出侧重 |
|------|---------------|----------|
| **CEO** | 合规前提 + 稳健增长 | 稳增长与兜底动作优先 |
| **控股股东** | 控制权/融资可得性 | 质押/融资/替代路径触发优先 |
| **机构投资者** | 风险预算 + 止损节奏 | 阈值、回撤、减仓执行条件优先 |
| **监管机构** | 合规性 + 系统性风险 | 违规预警、市场影响、处置建议优先 |
| **审计师** | 可验证性 + 内控有效性 | 证据链完整、内控缺陷识别优先 |
| **债权银行** | 偿债能力 + 担保安全 | 现金流预警、抵质押物价值监控优先 |

### 2.4 业务价值（验收口径）

| 指标 | 目标 | 说明 |
|------|------|------|
| 分析时间缩短 | 40% | 用"因果链 + 触发条件"替代纯文本研判 |
| 结论一致性 | >85% | 减少人工解释成本并提升一致性 |
| 误报率下降 | 30% | 以证据覆盖度/置信度约束推断结果 |
| 审计通过率 | 100% | 提升可审计性 |

## 三、研究内容与关键问题梳理

### 3.1 总体研究目标

构建面向「领域风险分析」的多智能体系统（以上市公司为案例），实现从多源数据到持续进化的闭环。

### 3.2 五体智能体：研究内容、关键问题与实现对照

#### （1）感知体（数据 → 事件）

| 维度 | 内容 |
|------|------|
| **研究内容** | 多源接入（监管披露/公告/财务与经营报表、新闻舆情、交易/运营数据等）；事件抽取（时间/主体/行为/对象/影响等结构化）；噪声过滤与可信度评估 |
| **关键问题** | 非结构化文本中「风险事件」的稳定抽取（⭐⭐⭐）；多源冲突（真伪/偏见）（⭐⭐⭐）；实时性与成本平衡（⭐⭐） |
| **实现进展** | ✅ 多源搜索（scout_cluster）、信息质量评估（alert_sentinel）、地理抽取（geo_extractor）已落地 |
| **待加强** | 五元组级事件抽取、可信度加权融合、增量更新与缓存 |

#### （2）认知体（事件 → 因果）

| 维度 | 内容 |
|------|------|
| **研究内容** | 事件对齐与实体消歧；动态事件图谱（Event Graph）；因果表示与推断（因果图、干预与识别等） |
| **关键问题** | 从相关性中识别因果（⭐⭐⭐⭐⭐，核心壁垒）；可解释风险传导链（⭐⭐⭐⭐）；外部知识（政策/行业）增强因果可信度（⭐⭐⭐） |
| **实现进展** | ✅ 因果发现引擎、SCM 建模、推断引擎（含 do-calculus、反事实）、风险传导路径发现、事件图谱可视化（EventGraph）已落地 |
| **待加强** | 领域知识注入提升发现准确率、动态因果图与时序因果 |

#### （3）决策体（因果 → 推演）

| 维度 | 内容 |
|------|------|
| **研究内容** | 风险分级；情景推演（多路径）；政策与市场/时间窗口等约束建模 |
| **关键问题** | 抑制 LLM「幻觉式推演」（⭐⭐⭐⭐）；多路径而非单结论（⭐⭐⭐）；路径概率与触发条件量化（⭐⭐⭐） |
| **实现进展** | ✅ 场景推演（基准/乐观/悲观）、敏感性分析、行动建议、政策模拟器、数字人格与博弈仿真引擎已落地 |
| **核心机制** | 因果链约束 + 多场景对比 + 证据驱动概率修正 |

#### （4）行动体（推演 → 业务）

| 维度 | 内容 |
|------|------|
| **研究内容** | 分级预警；研判报告/简报自动生成；策略建议（授信、监控、处置等场景） |
| **关键问题** | 推理结果转化为可执行指令（⭐⭐⭐）；输出合规与审计（⭐⭐⭐）；人机协同（确认与修订）（⭐⭐⭐） |
| **实现进展** | ✅ 红线/预警控制节点、报告 API、多格式导出、知识图谱展示与编辑已落地 |
| **预警分级** | Low / Medium / High / Critical（可对应绿/黄/橙/红四级运营语义） |

#### （5）进化体（反馈 → 优化）

| 维度 | 内容 |
|------|------|
| **研究内容** | 人工反馈与策略迭代；风险规则与权重调整；模型与知识库的持续优化 |
| **关键问题** | 专家经验结构化（⭐⭐⭐⭐）；避免「越学越偏」（⭐⭐⭐）；长期评估（准确率、提前量等）（⭐⭐⭐） |
| **实现进展** | ✅ 案例存储、记忆/复盘节点、知识复用服务、向量嵌入检索已落地，形成「分析—沉淀—检索—复用」闭环 |
| **与人对齐表述** | 文稿中的 RLHF 等可作为中远期增强方向，与当前「案例 + 向量 + 反馈入库」并存表述 |
