# 个性化响应模板系统设计

## 1. 设计理念

### 核心目标

```
用户反馈 → 偏好学习 → 模板调整 → 个性化响应
    ↓          ↓           ↓           ↓
  显式点赞   场景偏好     权重更新     自动适配
  隐式行为   模板偏好     参数优化     动态选择
```

### 平衡原则

| 原则 | 说明 |
|------|------|
| **平台兜底** | 新用户/冷启动使用平台默认模板 |
| **渐进个性化** | 随着反馈积累，逐步增强个性化 |
| **可控性** | 用户可手动调整/重置偏好 |
| **可解释** | 告知用户为何选择此模板 |

### 3.2 权重融合算法

```typescript
/**
 * 融合平台权重和用户权重
 */
function mergeTemplateWeights(
  platformWeights: Record<string, number>,  // 平台默认权重
  userWeights: Record<string, number>,      // 用户偏好权重
  confidence: number,                        // 用户偏好置信度
  options: {
    minPlatformWeight: number;   // 平台权重最小值（保证多样性）
    maxUserInfluence: number;    // 用户最大影响力
  }
): Record<string, number> {
  const merged: Record<string, number> = {};
  
  for (const templateId of Object.keys(platformWeights)) {
    const platformWeight = platformWeights[templateId] || 0.5;
    const userWeight = userWeights[templateId] ?? platformWeight;
    
    // 根据置信度平衡
    const userInfluence = Math.min(confidence, options.maxUserInfluence);
    const platformInfluence = 1 - userInfluence;
    
    // 确保平台权重不低于阈值
    merged[templateId] = Math.max(
      platformWeight * platformInfluence + userWeight * userInfluence,
      options.minPlatformWeight
    );
  }
  
  return normalizeWeights(merged);
}
```

### 4.2 学习算法

```typescript
/**
 * 偏好学习器
 */
class PreferenceLearner {
  /**
   * 从反馈中学习
   */
  learnFromFeedback(
    preference: UserTemplatePreference,
    feedback: FeedbackSignal
  ): UserTemplatePreference {
    const { templateId, queryType } = feedback.context;
    const scenario = preference.scenarioPreferences[queryType] || this.createDefaultScenario();
    
    // 1. 计算奖励分数
    const reward = this.calculateReward(feedback);
    
    // 2. 更新模板权重（使用 Bandit 算法）
    scenario.templateWeights[templateId] = this.updateWeight(
      scenario.templateWeights[templateId],
      reward,
      preference.learningState.sampleSize
    );
    
    // 3. 更新统计
    scenario.stats.usageCount++;
    if (feedback.explicit?.type === 'like') {
      scenario.stats.positiveFeedbackRate = this.updateRate(
        scenario.stats.positiveFeedbackRate,
        true,
        scenario.stats.usageCount
      );
    }
    
    // 4. 更新学习状态
    preference.learningState.sampleSize++;
    preference.learningState.confidence = this.calculateConfidence(
      preference.learningState.sampleSize,
      scenario.stats.positiveFeedbackRate
    );
    preference.learningState.lastUpdated = Date.now();
    
    // 5. 记录历史
    preference.feedbackHistory.push({
      timestamp: feedback.timestamp,
      query: feedback.query,
      queryType,
      templateId,
      feedback: feedback.explicit?.type || 'neutral',
      dwellTime: feedback.implicit?.dwellTime || 0,
      clickedItems: feedback.implicit?.clickedItems || 0,
    });
    
    return preference;
  }
  
  /**
   * 计算奖励分数（综合显式和隐式信号）
   */
  private calculateReward(feedback: FeedbackSignal): number {
    let reward = 0;
    
    // 显式反馈（权重 0.6）
    if (feedback.explicit) {
      const explicitReward = {
        'like': 1.0,
        'neutral': 0.5,
        'dislike': -1.0,
      };
      reward += explicitReward[feedback.explicit.type] * 0.6;
    }
    
    // 隐式行为（权重 0.4）
    if (feedback.implicit) {
      const { dwellTime, clickedItems, bounced, expandedDetails } = feedback.implicit;
      
      // 停留时间奖励（标准化）
      const dwellReward = Math.min(dwellTime / 60, 1);  // 60秒为满分
      
      // 点击奖励
      const clickReward = Math.min(clickedItems / 3, 1);  // 3次点击为满分
      
      // 展开奖励
      const expandReward = expandedDetails ? 0.5 : 0;
      
      // 惩罚快速离开
      const bouncePenalty = bounced ? -0.5 : 0;
      
      reward += (dwellReward * 0.4 + clickReward * 0.3 + expandReward * 0.2 + bouncePenalty) * 0.4;
    }
    
    return Math.max(-1, Math.min(1, reward));  // 归一化到 [-1, 1]
  }
  
  /**
   * 更新权重（使用衰减学习率）
   */
  private updateWeight(
    currentWeight: number,
    reward: number,
    sampleSize: number
  ): number {
    // 衰减学习率：样本越多，变化越小
    const learningRate = 0.1 / (1 + sampleSize * 0.01);
    
    // 向奖励方向调整
    const newWeight = currentWeight + learningRate * reward;
    
    return Math.max(0.1, Math.min(0.9, newWeight));  // 限制范围
  }
  
  /**
   * 计算偏好置信度
   */
  private calculateConfidence(
    sampleSize: number,
    positiveRate: number
  ): number {
    // 基于样本量和一致性计算置信度
    const sizeConfidence = Math.min(sampleSize / 50, 1);  // 50个样本达到满置信
    const consistencyConfidence = Math.abs(positiveRate - 0.5) * 2;  // 偏好一致性
    
    return sizeConfidence * 0.6 + consistencyConfidence * 0.4;
  }
}
```

## 5. 平衡策略

### 5.1 四象限平衡模型

```
              用户偏好强
                  │
                  │    ┌─────────────┐
                  │    │ 个性化模板   │
                  │    │ 优先用户     │
    用户自定义 ───┼────┤ 权重: 用户高 │
    （手动设置）  │    │ 置信度: 高   │
                  │    └─────────────┘
                  │
──────────────────┼──────────────────
                  │
                  │    ┌─────────────┐
                  │    │ 学习中      │
                  │    │ 混合权重     │
    用户偏好弱 ───┼────┤ 平衡平台/用户│
    （自动学习）  │    │ 置信度: 中   │
                  │    └─────────────┘
                  │
              平台默认强
```

### 5.2 动态调整规则

```typescript
/**
 * 平衡策略配置
 */
interface BalanceStrategy {
  // 最小平台权重（保证多样性）
  minPlatformWeight: number;  // 默认 0.2
  
  // 用户最大影响力
  maxUserInfluence: number;   // 默认 0.7
  
  // 冷启动阈值
  coldStartThreshold: number; // 默认 10 次反馈
  
  // 探索率（保持探索新模板）
  explorationRate: number;    // 默认 0.1
  
  // 置信度阈值
  confidenceThreshold: number; // 默认 0.6
}

/**
 * 应用平衡策略
 */
function applyBalanceStrategy(
  templateId: string,
  userPreference: UserTemplatePreference,
  strategy: BalanceStrategy
): {
  templateId: string;
  config: TemplateConfig;
  source: 'user' | 'platform' | 'mixed';
  confidence: number;
} {
  const { learningState, scenarioPreferences, global } = userPreference;
  
  // 1. 冷启动检查
  if (learningState.sampleSize < strategy.coldStartThreshold) {
    return {
      templateId,
      config: getPlatformTemplate(templateId),
      source: 'platform',
      confidence: 0,
    };
  }
  
  // 2. 探索检查（以一定概率尝试新模板）
  if (Math.random() < strategy.explorationRate) {
    const alternativeTemplate = selectAlternativeTemplate(templateId);
    return {
      templateId: alternativeTemplate,
      config: getPlatformTemplate(alternativeTemplate),
      source: 'platform',
      confidence: 0,
      reason: '探索新模板',
    };
  }
  
  // 3. 置信度检查
  if (learningState.confidence < strategy.confidenceThreshold) {
    // 混合模式：平台权重较高
    const mixedConfig = mergeConfigs(
      getPlatformTemplate(templateId),
      getUserTemplateConfig(templateId, scenarioPreferences),
      strategy.minPlatformWeight + 0.3  // 平台权重更高
    );
    return {
      templateId,
      config: mixedConfig,
      source: 'mixed',
      confidence: learningState.confidence,
    };
  }
  
  // 4. 高置信度：用户偏好优先
  const userConfig = getUserTemplateConfig(templateId, scenarioPreferences);
  const mergedConfig = mergeConfigs(
    getPlatformTemplate(templateId),
    userConfig,
    1 - strategy.maxUserInfluence  // 平台权重降低
  );
  
  return {
    templateId,
    config: mergedConfig,
    source: 'user',
    confidence: learningState.confidence,
  };
}
```

### 5.3 冲突处理

```typescript
/**
 * 冲突场景与处理
 */
const conflictHandlers = {
  // 场景1：用户偏好与最佳实践冲突
  'user_vs_best_practice': {
    detect: (userPref: any, queryType: string) => {
      // 例如：对比型查询用户选择列表模板
      return queryType === 'comparison_analysis' && 
             userPref.preferredLayout !== 'comparison';
    },
    resolve: (userPref: any, platform: any) => {
      // 策略：尊重用户选择，但添加提示
      return {
        useUser: true,
        warning: '您选择了非推荐模板，对比结果可能不够直观',
        allowSwitch: true,  // 允许切换到推荐模板
      };
    },
  },
  
  // 场景2：用户偏好与设备限制冲突
  'user_vs_device': {
    detect: (userPref: any, device: string) => {
      // 例如：移动设备选择了详细展示
      return device === 'mobile' && 
             userPref.density === 'detailed';
    },
    resolve: (userPref: any, device: string) => {
      // 策略：设备优先，但保留用户其他偏好
      return {
        density: 'normal',
        otherPrefs: userPref,
        reason: '移动设备已优化展示密度',
      };
    },
  },
  
  // 场景3：新模板发布
  'new_template': {
    detect: (userPref: any, newTemplateId: string) => {
      return !userPref.scenarioPreferences[newTemplateId];
    },
    resolve: (userPref: any, newTemplateId: string) => {
      // 策略：引导用户尝试新模板
      return {
        showNewTemplateHint: true,
        newTemplateId,
        message: '发现新的展示模板，是否尝试？',
      };
    },
  },
};
```

## 10. 总结

### 设计要点

1. **三层选择机制**：用户 > 群体 > 平台
2. **渐进个性化**：从平台默认逐步过渡到个性化
3. **平衡策略**：保证多样性和探索性
4. **用户可控**：提供手动调整和重置能力
5. **持续学习**：从反馈中自动优化

### 权重分配建议

| 场景 | 平台权重 | 用户权重 | 说明 |
|------|---------|---------|------|
| 冷启动（<10次） | 100% | 0% | 使用平台默认 |
| 学习中（10-50次） | 50% | 50% | 混合模式 |
| 成熟（>50次，高置信） | 20% | 80% | 个性化优先 |
| 手动设置 | 0% | 100% | 完全尊重用户 |
