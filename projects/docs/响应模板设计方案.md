# 响应模板设计方案

## 1. 设计理念

```
用户问题 → 意图识别 → 模板匹配 → LLM填充 → 渲染输出
              ↓           ↓           ↓          ↓
         "对比A和B"   匹配对比模板   生成结构化数据   对比卡片展示
```

**核心优势**：
- ✅ 展示格式可配置，无需改代码
- ✅ 支持A/B测试不同展示效果
- ✅ 业务人员可调整模板
- ✅ 模板与逻辑分离，易于维护

## 2. 预置模板列表

| 模板ID | 名称 | 适用场景 | 渲染组件 |
|--------|------|----------|----------|
| `comparison_analysis` | 对比分析 | A vs B 对比 | ComparisonCard |
| `ranking_list` | 排序列表 | 多选项排名 | RankingTable |
| `single_answer` | 单一答案 | 问答型查询 | AnswerCard |
| `recommendation_items` | 推荐列表 | 商品/内容推荐 | ItemList |
| `clarification` | 追问引导 | 信息不足 | ClarificationForm |
| `pros_cons` | 优缺点分析 | 决策辅助 | ProsConsCard |
| `step_by_step` | 步骤指南 | 操作指南 | StepGuideCard |
| `data_insight` | 数据洞察 | 数据分析 | InsightCard |

## 4. 模板选择逻辑

```typescript
/**
 * 模板选择器
 */
export class TemplateSelector {
  private templates: Map<string, ResponseTemplate>;
  
  constructor() {
    this.templates = new Map();
    this.registerDefaultTemplates();
  }
  
  /**
   * 选择最佳模板
   */
  selectTemplate(query: string, intentResult: IntentResult): ResponseTemplate {
    const candidates: Array<{ template: ResponseTemplate; score: number }> = [];
    
    for (const template of this.templates.values()) {
      const score = this.calculateMatchScore(query, intentResult, template);
      if (score > 0.5) {
        candidates.push({ template, score });
      }
    }
    
    // 按分数排序，返回最佳匹配
    candidates.sort((a, b) => b.score - a.score);
    
    if (candidates.length === 0) {
      // 兜底：使用默认推荐模板
      return this.templates.get('recommendation_items')!;
    }
    
    return candidates[0].template;
  }
  
  /**
   * 计算匹配分数
   */
  private calculateMatchScore(
    query: string,
    intent: IntentResult,
    template: ResponseTemplate
  ): number {
    let score = 0;
    
    // 1. 意图类型匹配 (权重 0.4)
    if (template.triggers.intentTypes.includes(intent.type)) {
      score += 0.4;
    }
    
    // 2. 问题模式匹配 (权重 0.3)
    for (const pattern of template.triggers.queryPatterns) {
      if (this.matchPattern(query, pattern)) {
        score += 0.3;
        break;
      }
    }
    
    // 3. 实体数量匹配 (权重 0.3)
    if (template.triggers.entityCount) {
      const entityCount = intent.entities?.length || 0;
      const { min, max } = template.triggers.entityCount;
      if (entityCount >= min && entityCount <= max) {
        score += 0.3;
      }
    }
    
    return score;
  }
  
  /**
   * 模式匹配
   */
  private matchPattern(query: string, pattern: string): boolean {
    // 将模式转换为正则表达式
    const regex = pattern
      .replace(/{(\w+)}/g, '(.+?)')
      .replace(/\s+/g, '\\s+');
    
    return new RegExp(regex, 'i').test(query);
  }
}
```

## 5. 与现有架构整合

```typescript
// 在 AgentRecommendationService 中使用模板

export class AgentRecommendationService {
  private templateSelector: TemplateSelector;
  
  async generateRecommendations(query: string, context: RecommendationContext) {
    // 1. 意图分析
    const intentResult = await this.runIntentAnalyzer(query);
    
    // 2. 选择模板
    const template = this.templateSelector.selectTemplate(query, intentResult);
    console.log('[Template] Selected:', template.id, template.name);
    
    // 3. 获取数据
    const data = await this.gatherData(query, context, template);
    
    // 4. 使用模板生成响应
    const response = await this.renderWithTemplate(template, data);
    
    return {
      ...response,
      metadata: {
        templateId: template.id,
        templateName: template.name,
      },
    };
  }
  
  /**
   * 使用模板渲染响应
   */
  private async renderWithTemplate(
    template: ResponseTemplate,
    data: any
  ): Promise<any> {
    // 构建 Prompt
    const prompt = this.buildPrompt(template.promptTemplate, data);
    
    // 调用 LLM
    const llmResponse = await this.llmClient.chat({
      messages: [{ role: 'user', content: prompt }],
      responseFormat: { type: 'json_object' },
    });
    
    // 解析并验证输出
    const output = JSON.parse(llmResponse.content);
    this.validateOutput(output, template.outputSchema);
    
    return {
      ...output,
      _renderConfig: template.renderConfig,
    };
  }
}
```

  const Component = RENDER_COMPONENTS[response._renderConfig?.component || 'ItemList'];
  
  return (

      <Component data={response} config={response._renderConfig?.style} />
    </div>
  );
}

## 9. 优势总结

| 方面 | 当前实现 | 模板系统 |
|------|---------|---------|
| 修改展示格式 | ❌ 改代码 | ✅ 改配置 |
| A/B 测试 | ❌ 不支持 | ✅ 支持 |
| 业务人员参与 | ❌ 需要开发 | ✅ 可自行调整 |
| 输出一致性 | ⚠️ 依赖 LLM | ✅ 模板约束 |
| 可维护性 | ⚠️ 逻辑耦合 | ✅ 关注点分离 |
| 扩展性 | ❌ 硬编码 | ✅ 插件化 |

## 10. 注意事项

1. **Prompt 质量**：模板的核心是 Prompt 设计，需要精心调优
2. **Schema 验证**：严格验证 LLM 输出符合 Schema
3. **兜底机制**：模板匹配失败时，使用默认模板
4. **版本兼容**：模板更新时，考虑旧版本数据的兼容性
5. **性能监控**：监控模板渲染耗时和成功率
