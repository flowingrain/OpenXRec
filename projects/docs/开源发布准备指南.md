# 开源发布准备指南

## 📋 概述

本文档说明将智弈全域风险态势感知平台作为开源项目发布时，需要移除或修改的内容。

---

## 🔴 必须移除的内容（安全敏感）

### 1. 环境变量文件

| 文件 | 操作 | 原因 |
|------|------|------|
| `.env.local` | **删除** | 包含 Supabase URL、密钥和数据库密码 |

**敏感内容**：
```bash
# ❌ 必须删除
NEXT_PUBLIC_SUPABASE_URL=https://nzwqgcwdrtlgrznvaiso.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=sb_publishable_wJ1-kyHRPAvFBQwxDCJSWg_Cl6klOIC
SUPABASE_DB_URL=postgresql://postgres:lwRqXm0JXf8wISyv@db.nzwqgcwdrtlgrznvaiso.supabase.co:5432/postgres
```

**替换方案**：
创建 `.env.example` 文件，提供占位符：
```bash
# ✅ 使用占位符
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
SUPABASE_DB_URL=postgresql://postgres:password@db.your-project.supabase.co:5432/postgres
```

### 2. 用户数据

| 目录/文件 | 操作 | 原因 |
|-----------|------|------|
| `data/memory-store/` | **删除** | 包含用户会话和记忆数据 |

**数据类型**：
- 用户会话历史
- 记忆数据
- Agent 状态

**处理方式**：
```bash
# 删除整个 data 目录
rm -rf data/
```

**添加到 .gitignore**：
```gitignore
# 用户数据
data/
*.json
```

### 2. 模型名称配置

| 位置 | 当前值 | 替换为 |
|------|--------|--------|
| 所有源代码文件 | `doubao-seed-2-0-pro-260215` | `process.env.LLM_MODEL` |

**需要修改的文件数量**：约 **80+** 个文件

**主要修改位置**：
- `src/app/api/*/route.ts` - API 路由
- `src/lib/langgraph/*.ts` - LangGraph 节点
- `src/lib/causal/*.ts` - 因果推理引擎
- `src/lib/knowledge/*.ts` - 知识管理
- `src/lib/agent-framework.ts` - Agent 框架

**修改脚本**：

创建 `scripts/replace-model-names.ts`：
```typescript
import { glob } from 'glob';
import { readFileSync, writeFileSync } from 'fs';

const files = await glob('src/**/*.{ts,tsx}');

files.forEach(file => {
  const content = readFileSync(file, 'utf-8');
  const updated = content
    .replace(
      /model:\s*['"]doubao-seed-2-0-pro-260215['"]/g,
      "model: process.env.LLM_MODEL || 'gpt-4'"
    )
    .replace(
      /model:\s*config\.model\s*\|\|\s*['"]doubao-seed-2-0-pro-260215['"]/g,
      "model: config.model || process.env.LLM_MODEL || 'gpt-4'"
    );

  writeFileSync(file, updated);
  console.log(`Updated: ${file}`);
});

console.log(`✅ Updated ${files.length} files`);
```

**环境变量配置**：

在 `.env.example` 中添加：
```bash
# 大语言模型配置
# 支持的模型：gpt-4, gpt-3.5-turbo, claude-3, doubao-seed-2-0-pro-260215 等
LLM_MODEL=gpt-4

# Agent 任务调度模型（可选，默认使用 LLM_MODEL）
AGENT_SCHEDULER_MODEL=gpt-4
```

## 🔧 开源后的配置说明

### 用户需要自行配置的内容

1. **Supabase 配置**：
   - 创建 Supabase 项目
   - 执行数据库迁移脚本
   - 配置环境变量

2. **环境变量**：
   ```bash
   # Supabase 配置
   NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
   SUPABASE_DB_URL=postgresql://postgres:password@db.your-project.supabase.co:5432/postgres

   # 大语言模型配置
   # 支持的模型：gpt-4, gpt-3.5-turbo, claude-3, doubao-seed-2-0-pro-260215 等
   LLM_MODEL=gpt-4

   # Agent 任务调度模型（可选，默认使用 LLM_MODEL）
   AGENT_SCHEDULER_MODEL=gpt-4
   ```

3. **Coze SDK**：
   - Coze SDK 的认证信息由平台自动注入
   - 如果在其他平台部署，需要配置 Coze API 密钥

### 数据库迁移

提供数据库迁移脚本：

```bash
# 执行 Supabase 初始化脚本
psql -h db.your-project.supabase.co -U postgres -d postgres -f supabase/init.sql

# 执行向量嵌入迁移
psql -h db.your-project.supabase.co -U postgres -d postgres -f supabase/migrations/002_add_vector_embeddings.sql
```
