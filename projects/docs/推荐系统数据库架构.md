# 可解释推荐系统 - 数据库架构文档

## 概述

可解释推荐系统采用**完全独立的数据库架构**，不依赖或复用任何其他项目的数据库、知识库、向量库、案例库。本系统专门为推荐场景设计，包含完整的知识管理、向量搜索、案例追踪、场景配置和算法管理功能。

## 数据库设计原则

1. **独立性**：完全独立的表结构，与态势智联项目无任何依赖关系
2. **完整性**：涵盖推荐系统的所有核心功能模块
3. **可扩展性**：支持多种推荐算法和向量嵌入模型
4. **高性能**：使用 pgvector 实现高效的向量搜索
5. **可观测性**：完善的案例记录和效果追踪

## 核心表结构

### 1. 推荐知识库 (recommendation_knowledge)

**用途**：存储推荐系统相关的知识文档，支持语义搜索

**核心字段**：
- `id` - UUID 主键
- `title` - 知识标题
- `content` - 知识内容
- `knowledge_type` - 知识类型（通用、策略、领域、规则、解释、最佳实践）
- `tags` - 标签数组
- `embedding` - 向量嵌入（2048维）
- `status` - 状态（active, archived, deprecated）

**索引**：
- 标题、类型、标签索引
- HNSW 向量索引（用于语义搜索）

**初始化数据**：4条示例知识文档

---

### 2. 推荐对象库 (recommendation_items)

**用途**：存储可推荐的各种对象（知识、实体、报告、案例等）

**核心字段**：
- `id` - UUID 主键
- `item_id` - 外部对象唯一标识
- `item_type` - 对象类型（knowledge, entity, report, case, scenario, article, dataset）
- `title` - 对象标题
- `category` / `subcategory` - 分类
- `tags` / `keywords` - 标签和关键词
- `quality_score` - 质量分数（0-1）
- `popularity_score` - 热度分数
- `content_embedding` - 内容向量嵌入
- `status` - 状态（active, hidden, archived, deleted）

**统计字段**：
- `view_count` - 浏览次数
- `click_count` - 点击次数
- `like_count` - 点赞次数
- `avg_rating` - 平均评分

**索引**：
- item_id、类型、分类索引
- 质量和热度索引
- HNSW 向量索引

**用途**：
- 存储所有可推荐的物品
- 支持内容相似度搜索
- 追踪用户交互统计

---

### 3. 推荐向量库 (recommendation_embeddings)

**用途**：存储各类向量化数据，支持多种推荐算法

**核心字段**：
- `id` - UUID 主键
- `embedding_type` - 向量类型
  - `user` - 用户向量
  - `item` - 物品向量
  - `item_content` - 物品内容向量
  - `user_preference` - 用户偏好向量
  - `user_behavior` - 用户行为向量
  - `context` - 上下文向量
  - `interaction` - 交互向量
- `target_id` - 关联的对象ID
- `embedding` - 向量数据（2048维）
- `model_name` - 向量模型名称
- `update_frequency` - 更新频率
- `last_updated_at` - 最后更新时间

**索引**：
- 按类型、目标ID索引
- HNSW 向量索引（为用户和物品类型创建）

**用途**：
- 支持协同过滤（用户-用户、物品-物品相似度）
- 支持内容推荐（物品内容相似度）
- 支持混合推荐算法

---

### 4. 推荐案例库 (recommendation_cases)

**用途**：存储推荐案例和场景，支持效果评估和优化

**核心字段**：
- `id` - UUID 主键
- `case_id` - 案例唯一标识
- `case_name` - 案例名称
- `case_type` - 案例类型
- `scenario_id` - 场景ID
- `user_id` - 用户ID
- `algorithm` - 使用的算法
- `recommended_items` - 推荐的物品列表（JSONB）
- `recommendation_explanations` - 推荐解释（JSONB）
- `diversity_score` - 多样性分数
- `novelty_score` - 新颖性分数
- `relevance_score` - 相关性分数

**效果指标**：
- `impressions` - 展示次数
- `clicks` - 点击次数
- `conversions` - 转化次数
- `ctr` - 点击率
- `conversion_rate` - 转化率
- `dwell_time_ms` - 停留时间
- `business_value` - 业务价值
- `roi` - 投资回报率

**用户反馈**：
- `user_feedback` - 用户反馈（JSONB）
- `user_rating` - 用户评分（1-5）
- `user_comments` - 用户评论

**索引**：
- case_id、类型、场景、用户、算法索引
- CTR、执行时间索引
- 可复用案例索引

**用途**：
- 记录每次推荐的完整信息
- 追踪推荐效果
- 支持案例复现和优化
- 支持A/B测试分析

---

### 5. 推荐场景库 (recommendation_scenarios)

**用途**：定义和管理推荐场景

**核心字段**：
- `id` - UUID 主键
- `scenario_id` - 场景唯一标识
- `name` - 场景名称
- `description` - 场景描述
- `category` - 场景分类
- `priority` - 优先级
- `context` - 上下文配置（JSONB）
- `constraints` - 约束条件（JSONB）
- `filters` - 过滤条件（JSONB）
- `default_algorithm` - 默认算法
- `available_algorithms` - 可用算法列表
- `fallback_algorithm` - 降级算法

**推荐策略配置**：
- `min_diversity` - 最小多样性（0-1）
- `min_novelty` - 最小新颖性（0-1）
- `max_repeat_ratio` - 最大重复率
- `max_item_age_days` - 物品最大年龄（天）
- `refresh_interval_hours` - 刷新间隔（小时）

**目标指标**：
- `target_ctr` - 目标点击率
- `target_conversion_rate` - 目标转化率
- `target_dwell_time_ms` - 目标停留时间

**A/B测试**：
- `is_ab_test` - 是否A/B测试
- `test_group` - 测试组

**索引**：
- scenario_id、分类、状态、优先级索引
- A/B测试索引

**初始化数据**：4个预设场景
1. `homepage_discover` - 首页发现
2. `related_knowledge` - 相关知识推荐
3. `trending_items` - 热门趋势
4. `personalized_feed` - 个性化推荐

**核心字段**：

  - `llm_enhanced` - LLM增强推荐
  - `custom` - 自定义算法
- `description` - 算法描述
- `version` - 算法版本
- `default_params` - 默认参数（JSONB）
- `required_params` - 必需参数列表
- `optional_params` - 可选参数列表

**性能配置**：
- `enabled` - 是否启用
- `max_concurrent_requests` - 最大并发请求数
- `timeout_ms` - 超时时间（毫秒）

**质量配置**：
- `min_quality_score` - 最小质量分数

- `total_errors` - 总错误次数
- `avg_response_time_ms` - 平均响应时间
- `avg_ctr` - 平均点击率
- `avg_conversion_rate` - 平均转化率

**索引**：
- algorithm_id、类型、状态索引
- A/B测试索引

**初始化数据**：7个预设算法

7. `llm_enhanced` - LLM增强推荐

## 向量搜索功能

### pgvector 扩展

- **维度**：2048维
- **距离度量**：余弦相似度（vector_cosine_ops）
- **索引类型**：HNSW（m=16, ef_construction=64）

### 向量搜索函数

1. **`search_recommendation_knowledge()`**
   - 用途：知识库语义搜索
   - 参数：查询向量、阈值、数量、类型过滤、标签过滤
   - 返回：相关知识文档及相似度

2. **`search_similar_items()`**
   - 用途：推荐对象相似度搜索
   - 参数：查询向量、阈值、数量、类型过滤、分类过滤、最低质量分数
   - 返回：相似对象及其质量指标

3. **`find_similar_users()`**
   - 用途：查找相似用户（协同过滤）
   - 参数：用户向量、阈值、数量
   - 返回：相似用户ID及相似度

4. **`find_similar_items_by_embedding()`**
   - 用途：查找相似物品（基于物品向量）
   - 参数：物品向量、阈值、数量
   - 返回：相似物品ID及向量类型

## 辅助函数

### 1. `update_recommendation_knowledge_embedding()`
更新知识库文档的向量嵌入

### 2. `update_recommendation_item_embedding()`
更新推荐对象的向量嵌入

### 3. `upsert_user_embedding()`
插入或更新用户向量（支持自动更新频率统计）

### 4. `log_recommendation_case()`
记录推荐案例，自动更新场景和算法统计

## 统计视图

### 1. `recommendation_embedding_stats`
向量嵌入统计视图
- 按类型统计向量数量
- 计算向量化覆盖率
- 平均更新频率

### 2. `recommendation_item_stats`
推荐对象统计视图
- 按类型和分类统计对象数量
- 平均质量和热度分数
- 总浏览量、点击量、CTR

### 3. `recommendation_algorithm_performance`
推荐算法性能视图
- 调用次数、错误次数、错误率
- 平均响应时间
- 平均CTR和转化率

### 4. `recommendation_scenario_performance`
场景效果统计视图
- 场景推荐总数
- 成功/失败案例数
- 实际CTR和转化率

## 数据流图

```
用户交互 → user_interactions (用户系统)
    ↓
用户画像更新 → user_profiles
    ↓
生成用户向量 → recommendation_embeddings (type=user)
    ↓
推荐请求 → recommendation_scenarios
    ↓
选择算法 → recommendation_algorithms
    ↓
获取候选集 → recommendation_items
    ↓
向量搜索 → recommendation_embeddings
    ↓
生成推荐 → recommendation_cases
    ↓
记录结果 → recommendation_history (用户系统)
```

## 使用示例

### 1. 插入推荐对象

```sql
INSERT INTO recommendation_items (
  item_id, item_type, title, description, category, tags, quality_score
) VALUES (
  'KB001', 'knowledge', '推荐系统入门', '推荐系统基础知识', 'general',
  ARRAY['推荐系统', '入门'], 0.9
);
```

### 2. 更新向量嵌入

```sql
SELECT update_recommendation_item_embedding(
  (SELECT id FROM recommendation_items WHERE item_id = 'KB001'),
  '[0.1, 0.2, ...]'::vector(2048)
);
```

### 3. 搜索相似物品

```sql
SELECT * FROM search_similar_items(
  '[0.1, 0.2, ...]'::vector(2048),
  0.6,  -- 相似度阈值
  10,   -- 返回数量
  'knowledge'  -- 物品类型
);
```

### 4. 记录推荐案例

```sql
SELECT log_recommendation_case(
  'case_001',
  'user-uuid',
  'homepage_discover',
  'hybrid_weighted',
  '{"cf_weight": 0.4, "cb_weight": 0.4}'::jsonb,
  '[{"item_id": "KB001", "score": 0.9}]'::jsonb
);
```

## 性能优化建议

1. **向量搜索优化**：
   - 使用 HNSW 索引实现近似最近邻搜索
   - 调整 `m` 和 `ef_construction` 参数平衡精度和性能

2. **缓存策略**：
   - 算法结果缓存（由 `cache_ttl_seconds` 控制）
   - 热门物品预加载

3. **批量操作**：
   - 批量更新向量嵌入
   - 批量插入推荐对象

4. **分区策略**：
   - `recommendation_cases` 按时间分区（大规模场景）
   - `recommendation_embeddings` 按类型分区

## 与用户系统的集成

推荐系统与用户系统（003_add_user_system.sql）的关系：

| 用户系统表 | 推荐系统表 | 关联方式 |
|-----------|-----------|---------|
| `users` | `recommendation_cases` | 通过 `user_id` 关联 |
| `user_interactions` | `recommendation_embeddings` | 交互数据生成用户向量 |
| `user_profiles` | `recommendation_cases` | 用户画像影响推荐策略 |
| `recommendation_history` | `recommendation_cases` | 推荐历史与案例追踪 |

## 总结

可解释推荐系统的数据库架构具有以下特点：

✅ **完全独立**：不依赖其他项目的数据库
✅ **功能完整**：涵盖知识、对象、向量、案例、场景、算法六大模块
✅ **高性能**：使用 pgvector 实现高效的向量搜索
✅ **可观测**：完善的案例记录和效果追踪
✅ **可扩展**：支持多种推荐算法和向量嵌入模型
✅ **易于使用**：提供丰富的辅助函数和视图

该架构为推荐系统的开发、测试、部署和优化提供了坚实的基础设施支持。
