# 知识图谱动态类型与交互更新说明

## 1. 背景

本次更新聚焦两个问题：

- 类型表达不足：固定规范类型中 `其他` 信息含量低，图例与实际抽取结果不一致。
- 交互体验回退：节点拖拽、点击确认/编辑存在不稳定或阻断式弹窗问题。

目标是让知识图谱“动态类型可见、颜色可辨、交互可用”。

---

## 2. 动态类型体系

### 2.1 新增类型注册表（JSON）

- 文件：`config/knowledge-graph/type-registry.json`
- 用途：持续沉淀动态发现的实体/关系标签，不再依赖 TS 硬编码扩展。
- 结构（简化）：
  - `entityTypes[]`: `label` / `canonical` / `description` / `count` / `lastSeenAt`
  - `relationTypes[]`: 同上

### 2.2 读写逻辑

- 模块：`src/lib/knowledge-graph/type-registry.ts`
- 核心能力：
  - `loadTypeRegistry()`: 读取注册表
  - `updateTypeRegistry()`: 增量更新标签统计
  - `findCanonicalFromRegistry()`: 优先按注册表映射 canonical，失败再走归一化规则

### 2.3 抽取接口集成

- 接口：`src/app/api/knowledge-graph/extract/route.ts`
- 处理流程：
  1. 抽取前加载注册表
  2. LLM 返回动态类型 + 实体/关系
  3. 实体关系 canonical 映射时优先使用注册表
  4. 抽取后将动态类型与原始标签增量写回注册表

---

## 3. 类型与颜色展示策略

### 3.1 展示策略

- 组件：`src/components/knowledge/KnowledgeGraph.tsx`
- 图例展示动态类型（而非固定“前 N 个规范类型”）。
- 节点与边的文本优先展示动态原始标签（`typeLabelRaw` / `relationLabelRaw`）。

### 3.2 自动颜色区分

- 基于动态标签进行稳定哈希配色：
  - 同标签稳定同色
  - 不同标签自动区分
- 节点、边线、关系标签、图例颜色统一使用动态标签配色。

### 3.3 关于 `其他`

- 规范类型/关系展示中不再强调 `其他`。
- 映射兜底改为更有信息密度的语义类（实体偏向 `行业`，关系偏向 `关联`）。

---

## 4. 交互与可用性修复

### 4.1 拖拽回弹修复

- 仅在图谱结构（实体/关系集合）变化时重排布局，避免用户拖拽后被重置。

### 4.2 覆盖层拦截修复

- 图例/状态等 overlay 使用 `pointer-events-none`，只在按钮区域开启 `pointer-events-auto`。
- 防止 ReactFlow 画布被覆盖层“吃掉事件”，表现为“像 PNG 一样不能拖拽”。

### 4.3 点击反馈与快捷操作恢复

- 点击节点/边显示选中详情卡片。
- 支持快捷 `编辑` / `确认` 操作。

### 4.4 去阻断弹窗

- 知识图谱确认/编辑流程不再使用浏览器 `alert`。
- 改为非阻断提示，减少流程中断。

---

## 5. 增量合并一致性修复

问题：实体按名称去重后，关系仍引用临时 ID，导致关系端点悬挂，表现为节点或关系“减少/不显示”。

修复：

- 在 `OpenXRecHome` 的知识图谱增量合并里，引入“名称 -> 稳定实体 ID”重映射。
- 关系端点在合并时统一映射到稳定 ID，缺端点关系跳过，避免污染布局。

---

## 6. 已知限制与后续建议

- 动态类型数量过多时，图例仍需做分组/折叠（可按频次 Top-N + 其余收纳）。
- 全项目仍有部分业务页面使用 `alert/confirm`，建议后续统一替换为非阻断 UI 提示组件。
